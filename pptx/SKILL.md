## 设计思路

**不要创建无聊的幻灯片。** 白色背景上的普通要点不会给人留下深刻印象。考虑为每张幻灯片使用列表中的想法。

### 开始之前

- **选择大胆、与内容相关的配色方案**：调色板应该感觉是为这个特定主题设计的。如果将你的颜色交换到完全不同的演示文稿中仍然"有效"，说明你的选择不够具体。
- **主次分明**：一种颜色应该占主导地位（60-70% 的视觉权重），配合 1-2 种支持色调和一种鲜明的强调色。永远不要给所有颜色相同的权重。
- **深浅对比**：标题和结论幻灯片使用深色背景，内容幻灯片使用浅色（"三明治"结构）。或者全程使用深色以营造高级感。
- **坚持视觉母题**：选择一个独特的元素并重复使用——圆角图片框、彩色圆圈中的图标、厚单侧边框。在每张幻灯片中都使用它。

### 配色方案

选择与主题匹配的颜色——不要默认使用通用蓝色。使用这些配色方案作为灵感：

| 主题           | 主色                | 辅色              | 强调色              |
| -------------- | ------------------- | ----------------- | ------------------- |
| **午夜商务**   | `1E2761` (深蓝)     | `CADCFC` (冰蓝)   | `FFFFFF` (白色)     |
| **森林与苔藓** | `2C5F2D` (森林绿)   | `97BC62` (苔藓绿) | `F5F5F5` (奶油色)   |
| **珊瑚活力**   | `F96167` (珊瑚橙)   | `F9E795` (金色)   | `2F3C7E` (深蓝)     |
| **暖赤陶**     | `B85042` (赤陶色)   | `E7E8D1` (沙色)   | `A7BEAE` (鼠尾草绿) |

### 每张幻灯片

**每张幻灯片都需要视觉元素**——图像、图表、图标或形状。纯文本幻灯片容易被遗忘。

**布局选项：**

- 双栏（左侧文本，右侧插图）
- 图标 + 文本行（彩色圆圈中的图标，粗体标题，下方描述）
- 2x2 或 2x3 网格（一侧图像，另一侧内容块网格）
- 半出血图像（左侧或右侧完整）带有内容叠加

**数据展示：**

- 大数据展示（大数字 60-72pt，下方小标签）
- 对比列（之前/之后、优/缺、并排选项）
- 时间线或流程图（编号步骤，箭头）

**视觉修饰：**

- 章节标题旁的小彩色圆圈中的图标
- 关键数据或口号的斜体强调文本

### 字体排印

**选择有趣的字体组合**——不要默认使用 Arial。选择有个性的标题字体，并配以干净的正文字体。

| 标题字体       | 正文字体         |
| -------------- | ---------------- |
| 微软雅黑       | 微软雅黑 Light   |
| 思源黑体 Heavy | 思源黑体 Regular |
| 微软正黑       | 微软正黑 Light   |
| 方正黑体       | 思源宋体         |
| 华文行楷       | 微软雅黑         |
| 站酷快乐体     | 方正书宋         |
| Consolas       | 思源等宽         |

| 元素       | 大小         |
| ---------- | ------------ |
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题   | 20-24pt 粗体 |
| 正文文本   | 14-16pt      |
| 说明文字   | 10-12pt 柔和 |

### 间距

- 最小边距 0.5"（约 1.27cm）
- 内容块之间 0.3-0.5"（约 0.76-1.27cm）
- 留出呼吸空间——不要填满每一英寸

### 避免（常见错误）

- **不要重复相同的布局**——在幻灯片之间变化列、卡片和展示框
- **不要居中对齐正文**——左对齐段落和列表；仅居中标题
- **不要吝啬大小对比**——标题需要 36pt+ 才能从 14-16pt 正文中脱颖而出
- **不要默认使用蓝色**——选择反映特定主题的颜色
- **不要随机混合间距**——选择 0.3" 或 0.5" 间隙并一致使用
- **不要只设计一张幻灯片而保持其他幻灯片简陋**——要么完全投入，要么全程保持简洁
- **不要创建纯文本幻灯片**——添加图像、图标、图表或视觉元素；避免纯标题 + 要点
- **不要忘记文本框填充**——当对齐线条或形状与文本边缘时，在文本框上设置 `margin: 0` 或偏移形状以考虑填充
- **不要使用低对比度元素**——图标和文本都需要与背景形成强烈对比；避免浅色文本配浅色背景或深色文本配深色背景
- **永远不要在标题下使用强调线**——这些是 AI 生成幻灯片的标志；改用留白或背景色

## 质量保证（必需）

**假设存在问题。你的工作是找到它们。**

你的第一次渲染几乎从不正确。将 QA 视为错误狩猎，而不是确认步骤。如果你在首次检查中发现了零问题，说明你看得不够仔细。

### 内容质量保证

```bash
python -m markitdown output.pptx
```

检查缺失内容、拼写错误、错误顺序。

**使用模板时，检查剩余的占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout|占位|示例|此处"
```

如果 grep 返回结果，请在声明成功之前修复它们。

### 视觉质量保证

**⚠️ 使用子代理**——即使是 2-3 张幻灯片。你一直盯着代码，会看到你期望的东西，而不是实际存在的东西。子代理有全新的视角。

将幻灯片转换为图像（参见 [转换为图像](#转换为图像)），然后使用此提示词：

```text
视觉上检查这些幻灯片。假设存在问题——找到它们。

查找：
- 重叠元素（文本穿过形状、线条穿过文字、堆叠元素）
- 文本溢出或在边缘/框边界处被截断
- 为单行文本定位的装饰线条但标题换行为两行
- 来源引用或页脚与上方内容冲突
- 元素太近（< 0.3" 间隙）或卡片/章节几乎接触
- 间隙不均匀（一个地方有大空隙，另一个地方拥挤）
- 距幻灯片边缘边距不足（< 0.5"）
- 列或类似元素未对齐一致
- 低对比度文本（例如，奶油色背景上的浅灰色文本）
- 低对比度图标（例如，没有对比圆圈的深色背景上的深色图标）
- 文本框太窄导致过度换行
- 剩余的占位符内容

对于每张幻灯片，列出问题或关注领域，即使是次要的。

阅读并分析这些图像：
1. /path/to/slide-01.jpg（预期：[简要描述]）
2. /path/to/slide-02.jpg（预期：[简要描述]）

报告发现的所有问题，包括次要问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图像 → 检查
2. **列出发现的问题**（如果未发现问题，更批判性地再次查看）
3. 修复问题
4. **重新验证受影响的幻灯片**——一个修复通常会创建另一个问题
5. 重复直到完整遍历未发现新问题

**在完成至少一次修复和验证循环之前，不要声明成功。**

## 转换为图像

将演示文稿转换为单个幻灯片图像以进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这将创建 `slide-01.jpg`、`slide-02.jpg` 等。

修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 依赖项

- `pip install "markitdown[pptx]"` - 文本提取
